## 3. 策略设计

### 3.1 核心逻辑

X5 = X2-A（分段降仓基线）+ Dual Momentum 绝对动量过滤层

```
信号层（与X2-A完全一致）:
  F1 = tanh(ratio_dev × 30) × 0.5   # 比价MA20方向
  F2 = accel_diff.clip(±0.02) × 5.0 # 动量加速度
  style_score = F1 + F2
  candidate_g = style_score > 0     # 二元方向（相对动量）

择时层（X2-A分段降仓 + Dual Momentum绝对动量）:
  # 趋势防御（X2-A原版）
  MA50双跌破 → weight=0.50
  MA75双跌破 → weight=0.25
  
  # 动量防御（X5新增）
  g_12m_mom = g_close.pct_change(252).shift(1)
  v_12m_mom = v_close.pct_change(252).shift(1)
  both_neg_mom = (g_12m_mom < 0) & (v_12m_mom < 0)
  if both_neg_mom:
      weight = min(weight, 0.25)
  
  # 方向始终由 style_score 决定（X2-A修复，保留）
  signal = 'growth' if candidate_g else 'value'
```

### 3.2 关键参数

| 参数 | 值 | 说明 |
|:----:|:----:|:----|
| MOM_LOOKBACK | 252 | 12月动量回看期（Antonacci 原始论文）|
| ABS_MOM_CAP | 0.25 | 绝对动量降仓上限（与 MA75 一致）|
| MOM_THRESHOLD | 0.0 | 动量阈值（<0 即负动量）|

### 3.3 设计原则

1. 不改变 F1/F2 和 X2-A 任何参数（因子要少）
2. Dual Momentum 独立择时层，不干扰信号层
3. 双双负动量（符合 Antonacci 原始定义）
4. ABS_MOM_CAP=0.25 与 MA75 降仓一致（避免双重降仓冲突）
5. 252 日回看期（Antonacci 原始论文）

## 4. 失败原因分析

### 4.1 核心失败根因：252 日动量是深度滞后指标

### 4.2 关键熊市时段触发情况

| 熊市时段 | 触发周数/总周数 | 评价 |
|:----|:----:|:----|
| 2015股灾 (2015-06~09) | 0/13 | ✗ 完全失效 |

| 2022熊市 (2022-01~10) | 2/38 | ✗ 太晚 |
| 2025调整 | 0/48 | ✗ 完全失效 |

### 4.3 三大失败根因

1. **252 日动量是深度滞后指标**：2015 股灾前是牛市，252 日动量都是正的，即使股灾发生，252 日回看期太长，动量还没转负
2. **2016/2018/2019 震荡市频繁误触发**：这些时段并非真正危险，但 Dual Momentum 频繁降仓，错失反弹（whipsaw 效应）
3. **与 MA75 降仓功能重叠但更差**：MA75 是趋势防御，Dual Momentum 是动量防御，但绝对动量滞后于趋势，没有提供领先信号

### 4.4 MA75 vs Dual Momentum 叠加分析

- 两者同时触发：16 周
- 仅 Dual Momentum 触发：39 周（多为震荡市误触发）
- 仅 MA75 触发：48 周
- Dual Momentum 触发中与 MA75 重叠的比例：29.1%
- → Dual Momentum 多为独立触发，但独立触发时段往往是震荡市而非真正的熊市

## 9. 教训总结

1. **绝对动量不适合 A 股牛市急跌**：2015 股灾 0 触发是结构性失效，不是参数问题
2. **单一 lookback 框架已死**：252 日太滞后，126 日稍好但年化代价太大，cap=0 清仓灾难性恶化
3. **绝对动量与 MA75 功能重叠但更差**：MA75 是趋势防御（已跌破确认），绝对动量是动量防御（滞后于趋势）
4. **震荡市 whipsaw 是最致命陷阱**：2016/2018/2019 误触发 49 周，错失反弹
5. **进攻优先于防御**：X2 瓶颈是收益提升而非回撤控制，X6 应走进攻方向（都涨超配成长）
