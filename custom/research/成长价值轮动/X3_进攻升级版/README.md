## 三、优化内容

X3 在 X2 基础上尝试四项进攻升级：

1. **F1 升级为 KST 多周期动量**（替代比价 MA20 方向）
   ```
   ratio = (g_close/v_close).shift(1)
   roc10/15/20/30 = ratio.pct_change(10/15/20/30)
   kst = roc10*1 + roc15*2 + roc20*3 + roc30*4
   kst_std = kst.rolling(252).std()
   f1 = tanh(kst / kst_std) * 0.5
   ```
2. **style_score 连续加权**（替代二元切换）
   - `style_score > 0.3` → `w_g=0.65`（都涨时波动率加成，上限 0.75）
   - `style_score < -0.3` → `w_g=0.35`
   - 其他 → `w_g=0.5`
3. **ADX 过滤 F2**（震荡市 ADX<20 时 F2 置 0）
   - ADX 用 close 近似计算（无 high/low），Wilder smoothing，period=14
4. **F2 clip 放宽测试**（±0.02 vs ±0.025）

降仓逻辑保持 X2 方案 A（MA50→50%，MA75→25%）不变。

## 五、失败原因分析（三重错配）

X3 失败并非单点失误，而是**三重错配**叠加，每一项都在错误的环节做了错误的优化：

### 5.1 错配一：KST 用错对象（F1 升级失败，-0.77pp）

- **意图：** 用 KST 多周期动量替代比价 MA20 方向，捕捉多周期趋势
- **实际效果：** KST 信号平滑后**滞后**于原版 MA20 方向，且对成长/价值比价的转折点识别反而变慢
- **错配本质：** F1 原本是"比价偏离均值的方向判断"，KST 是"比价自身的动量"。两者目标对象不同——KST 适合用在**单一资产**的趋势识别，用在**比价**上反而模糊了 short-term 反转信号
- **教训：** 替换因子前必须验证因子与原因子是否针对同一对象、同一特征

### 5.2 错配二：连续加权分仓稀释方向性头寸（-11.43pp，主犯）

- **意图：** 都涨时通过 `w_g=0.65/0.35` 适度超配成长，避免二元全仓切换的踏空
- **实际效果：** 把 100% 成长 → 65% 成长 + 35% 价值，**稀释了方向性头寸**
- **错配本质：** 成长/价值轮动策略的 Alpha 来源是**单边方向性头寸**（100% 成长或 100% 价值）。连续加权把"全仓押一个方向"变成"65/35 分仓"，相当于把方向性收益换成 beta 收益，年化直接腰斩（38% → 19.94%）
- **教训：** 轮动策略的本质是"押方向"，任何"分仓平滑"的优化都可能稀释核心 Alpha；只有当**两资产长期同向**时连续加权才合理，但成长/价值本质是反向轮动

### 5.3 错配三：ADX 过滤错对象（F2 ADX 过滤失败，-0.79pp）

- **意图：** 强趋势市场才相信 F2（动量加速度），震荡市过滤噪声
- **实际效果：** ADX<20 时把 F2 整体置 0，结果**滤掉了 F2 的有效部分**
- **错配本质：** F2 是动量加速度（加速度变化），不是趋势方向。ADX 衡量的是趋势强度，对 F2 这种二阶信号没有过滤意义——震荡市里动量加速度依然可能有效（反转的前兆），而 ADX 过滤把它一刀切掉
- **教训：** 过滤器的对象必须与被过滤因子在同一维度；ADX 适合过滤 F1 这种趋势方向因子，不适合过滤 F2 这种加速度因子

### 5.4 唯一正贡献：clip ±0.025（+0.13pp）

- 放宽 clip 阈值让 F2 在极端加速度下有更大表达空间
- 改善幅度极小（+0.13pp），但**至少不是负贡献**
- **保留为 X4 唯一从 X3 继承的优化**

## 六、量化研究员评价结论

### 6.1 核心结论

**评价形式：** 2 位 Alpha 派研究员（追求最高收益）vs 1 位风控派研究员（控制夏普/回撤）

**核心结论：**
1. **X3 整体判定为失败版本**，年化从 38.00% 暴跌至 19.94%（-18.06pp），Sharpe 从 1.313 跌至 0.957
2. **失败根因是"三重错配"**：KST 用错对象 + 连续加权分仓稀释 + ADX 过滤错对象
3. **回撤改善（+3.46pp）不构成采纳理由**：这是连续加权稀释方向性头寸的副产品——收益也稀释了，Calmar 反而劣化（1.009 → 0.583）
4. 风控派研究员明确表态：**回撤改善若以收益腰斩为代价，是不可接受的**，违反"收益为先"原则
5. **X3 不可作为下一轮基准**，X4 必须**回退到 X2 基线**重新出发

### 6.2 三重错配的层次

- **主犯：** 连续加权（-11.43pp）—— 直接稀释方向性 Alpha
- **从犯：** ADX 过滤（-0.79pp）+ KST 升级（-0.77pp）—— 各自错配，叠加放大劣化
- **唯一亮点：** clip ±0.025（+0.13pp）—— 唯一正贡献，应被 X4 继承

## 七、X4 优化方向

基于 X3 失败教训，X4 应**回退 X2 基线**，仅保留 X3 的 clip 优化，并新增 RSRS 领先撤退层：

| 优先级 | 项目 | 说明 |
|:------:|:----:|:----:|
| P0 | 回退 X2 基线 | X4 起点 = X2 方案 A 基线，丢弃 X3 全部 KST/连续加权/ADX |
| P0 | 保留 clip ±0.025 | X3 唯一正贡献，作为 X4 的默认 F2 clip |
| P1 | **加 RSRS 领先撤退层** | 时变阻力支撑指标，z<-0.7 触发领先撤退（在 MA50/MA75 触发前提前降仓） |
| - | 不再做 KST 替换 | KST 用错对象，放弃 |
| - | 不再做连续加权 | 轮动策略 Alpha 来源于方向性头寸，禁止稀释 |
| - | 不再做 ADX 过滤 F2 | ADX 与 F2 不同维度，放弃；若需过滤可考虑用 ADX 过滤 F1 |

**X4 优化目标：**
- 在不损失 X2 年化（≥38%）的前提下，通过 RSRS 领先撤退降低回撤（目标 -32% ~ -35%）
- 提升 Calmar（目标 ≥1.1），Sharpe 不劣化

## 十、教训总结（供后续版本参考）

1. **轮动策略的 Alpha 来源是方向性头寸**，任何"分仓平滑/连续加权"都可能稀释核心收益
2. **替换因子前必须验证因子对象一致性**：KST 适合单一资产趋势，不适合比价方向
3. **过滤器维度必须与被过滤因子一致**：ADX（趋势强度）不适合过滤 F2（加速度二阶信号）
4. **消融分析是验证优化有效性的金标准**：通过逐项关闭组件可清晰识别正/负贡献，避免"整体看似改善实则被某项掩盖"
5. **回撤改善若以收益腰斩为代价不可接受**：Calmar = 年化/回撤，收益腰斩会导致 Calmar 同步劣化
