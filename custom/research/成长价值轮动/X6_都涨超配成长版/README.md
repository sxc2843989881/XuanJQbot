## 3. 策略设计

### 3.1 核心逻辑

X6 = X2-A（分段降仓基线）+ 都涨状态覆盖层

```
信号层（保持X2-A不变）:
  F1 = tanh(ratio_dev × 30) × 0.5        # 比价MA20方向
  F2 = accel_diff.clip(±0.02) × 5.0      # 动量加速度
  style_score = F1 + F2
  candidate_g = style_score > 0           # 二元方向（相对动量）

状态覆盖层（X6新增，仅都涨时生效）:
  roc_g60 = g_close.pct_change(60).shift(1)
  roc_v60 = v_close.pct_change(60).shift(1)
  both_up = (roc_g60 > 0) & (roc_v60 > 0)
  final_g = candidate_g | both_up         # 都涨时强制选成长（覆盖F1/F2）

择时层（保持X2-A不变）:
  MA50双跌破 → weight=0.50
  MA75双跌破 → weight=0.25
```

### 3.2 关键参数

| 参数 | 值 | 说明 |
|:----:|:----:|:----|
| ROC_WINDOW | 60 | 60日ROC窗口（A 股动量有效窗口1-3月上界）|
| ROC_THRESHOLD | 0.0 | 符号阈值法（>0即上涨，最简形式）|
| OVERWEIGHT_GROWTH | True | 都涨时强制选成长 |

## 4. 灾难性失败根因分析

### 4.1 三大失败根因

**根因1：统计平均≠每期成立（反模式1衍生）**
- A2研究员的洞察"都涨时成长超额+53pp"是2015/2019/2020三轮的平均
- 但2017蓝筹行情是反例：价值100大涨而成长100不涨
- 用平均超额掩盖分布方差，犯了"统计显著≠经济逻辑每期成立"的错误

**根因2：覆盖（override）而非叠加（overlay）破坏F1/F2（反模式15变种）**
- `final_g = candidate_g | both_up`强制覆盖19.8%的F1/F2决策
- 相当于砍掉19.8%的有效信号
- F1（比价MA20相对动量，噪声小）+ F2（动量加速度，捕捉拐点）是已验证的有效组合
- 被60日ROC的滞后绝对动量符号覆盖，信息维度倒退

**根因3：都涨率43.7%远超预期，"都涨超配成长"退化为"长期超配成长"**
- 当都涨是常态而非例外时，覆盖层实质上让策略退化为"长期持成长+择时降仓"
- 违背风格轮动初衷（应在成长/价值间动态切换）
