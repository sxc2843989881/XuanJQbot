## 三、优化内容

X4 在 X2 方案 A 基线上做三项变更：

1. **回退 X2 基线**（MA20-F1 + F2 + 二元切换 + 分段降仓）
   - F1：比价 MA20 方向（`f1=0.5`），与 X1/X2 完全相同
   - F2：动量加速度（`f2=5.0`），二元切换 + MA 分段降仓（MA50→50%，MA75→25%）
2. **clip ±0.025**（X3 唯一正贡献，放宽自 X2 原版 ±0.02）
   - 失败：相对 X2 基线 -1.11pp 年化（详见 4.2 节）
3. **RSRS 领先撤退层**（修正版 RSRS = β × R²）
   - N=18（OLS 窗口），M=600（标准化窗口）
   - 对每个时点用最近 N 日 (high, low) 做 OLS：`high = α + β×low + ε`
   - `rsrs = β × R²`，`z = (rsrs - rolling_mean(M)) / rolling_std(M)`
   - `z < -0.7 → rsrs_cut = 0.5`（中度降仓）
   - `z < -1.0 → rsrs_cut = 0.2`（重度降仓）
   - `z = NaN → rsrs_cut = 1.0`（早期段不触发）
   - 信号 `shift(1)` 防未来函数
   - 对成长/价值分别计算，取 `min(z_g, z_v)` 作为更保守信号
   - 多信号取最保守仓位：`final_position = min(ma_position, rsrs_cut)`
   - 失败：相对 X4-no-RSRS -5.35pp 年化（详见 4.3 节）

## 五、失败原因分析（数据问题为主 + 方法论误用为辅）

X4 失败与 X3 的"三重错配"（纯方法论失误）不同，**X4 的失败以数据问题为主因，方法论误用为辅**。

### 5.1 主因：RSRS 因伪 high/low 数据失效（数据问题）

- **数据现状：** 87.6% 时段（2012-12-31 ~ 2024-10-28）真实 high/low 为空，用 `max/min(close_t, close_{t-1})` 近似
- **失效机制：**
  1. 伪 high/low = `max/min(close_t, close_{t-1})`，本质是**相邻收盘价的极值**，丢失了日内真实波动幅度
  2. 用伪数据做 OLS 回归 `high = α + β×low + ε`，β 失去"阻力位斜率"的物理意义，退化为纯统计量
  3. 伪数据使 z-score 分布出现**肥尾**（fat tail），极端值频率远高于正态
- **直接证据：** RSRS 触发频率 36.8%（N=18）~ 41.3%（N=10），**远超理论值 15.87%**（z<-1.0 单尾正态概率）
  - 触发率 36.8% 意味着 z<-1.0 的实际频率是理论的 2.32 倍
  - 这不是策略设计问题，是 z-score 统计特性被伪数据破坏
- **后果：** RSRS 频繁误触发降仓（重度降仓 162 周 / 560 有效周 = 28.9%），把进攻头寸不断砍掉，年化被压 -5.35pp

### 5.2 辅因一：clip ±0.025 在进攻框架下压收益（方法论误用）

- X3 报告 clip ±0.025 是正贡献（+0.13pp），但那是**在 X3 收益已被连续加权腰斩（19.94%）的防守语境下**的相对改善
- X4 回退到 X2 进攻框架（38% 级别）后，放宽 clip 让 F2 在极端加速度下表达更大，反而**压低收益 -1.11pp**
- **教训：** X3 的"clip 正贡献"结论**不可迁移到进攻框架**，因子优化的结论高度依赖上下文

### 5.3 辅因二：RSRS 方法论误用对象（方法论误用）

- RSRS（阻力支撑相对强度）设计初衷是用于**单一资产**的支撑位识别
- X4 把它用于**成长/价值双指数**，并对两资产分别计算后取 `min(z_g, z_v)`
- `min` 聚合策略过于保守：任一资产 RSRS 偏低就触发降仓，放大了误触发概率
- z-mean 聚合（X4-mean）虽缓解但未根治，年化仍 -2.79pp 低于 X2

### 5.4 N=24 回撤最优但年化仍低于 X2

- N=24 时回撤 -32.09%（全方案最优），但年化 30.55% 仍低于 X2 基线 36.34%（-5.79pp）
- 说明即便参数调到回撤最优，RSRS 在伪数据下依然净拖累收益
- **不能仅凭回撤改善就采纳**——Calmar 0.952 仍低于 X2 的 0.965

## 六、量化研究员评价结论

### 6.1 评价形式

2 位 Alpha 派研究员（追求最高收益）vs 1 位风控派研究员（控制 Sharpe/回撤），2v1 讨论，以收益为先、风控为辅。

### 6.2 核心结论

1. **X4 整体判定为失败版本**，年化从 36.34% 跌至 29.88%（-6.46pp），Sharpe 从 1.271 跌至 1.187，Calmar 从 0.965 跌至 0.815
2. **失败根因分类：**
   - **主因（数据问题）：** RSRS 因 87.6% 时段用伪 high/low，z-score 肥尾，触发率 36.8% 远超理论 15.87%，频繁误降仓压收益 -5.35pp
   - **辅因（方法论误用）：** clip ±0.025 在进攻框架下压收益 -1.11pp（X3 结论不可迁移）；RSRS 用于双资产 + min 聚合过于保守
3. **回撤改善（+1.00pp）不构成采纳理由：**
   - 改善幅度 < 2pp 显著性阈值
   - 风控派研究员明确表态：回撤改善若以年化损失 5.35pp 为代价，权衡比 5.33 远超可接受范围，违反"收益为先"原则
4. **X4 不可作为下一轮基准**，X5 必须**回退到 X2 基线**重新出发
5. **RSRS 暂时放弃**，直到获取全周期真实 high/low 数据再评估

### 6.3 失败根因层次

- **主犯（数据问题）：** 伪 high/low 导致 RSRS z-score 肥尾（-5.35pp）
- **从犯（方法论）：** clip ±0.025 进攻框架下压收益（-1.11pp）
- **次要：** RSRS 双资产 min 聚合过保守（z-mean 缓解但仍 -2.79pp）

## 七、X5 优化方向

基于 X4 失败教训，X5 应**回退 X2 基线**，放弃 RSRS（数据受限），改走 Dual Momentum 绝对动量过滤：

| 优先级 | 项目 | 说明 |
|:------:|:----:|:----:|
| P0 | 回退 X2 基线 | X5 起点 = X2 方案 A 基线（clip±0.02，无 RSRS），丢弃 X4 全部 clip 放宽 + RSRS |
| P0 | 放弃 RSRS | 直到获取全周期真实 high/low 数据再评估；当前数据下 RSRS 净拖累 |
| P1 | **Dual Momentum 绝对动量过滤** | 用绝对动量（近 12 月收益 < 0 时降仓）替代 RSRS 作为撤退层，仅需 close 数据，不依赖 high/low |
| - | 不再做 clip 放宽 | X4 验证进攻框架下放宽 clip 压收益，保持 X2 原版 ±0.02 |
| - | 不再做 RSRS 双资产 min 聚合 | min 过保守，z-mean 仍失败，整体方法论在伪数据下不可用 |

**X5 优化目标：**
- 在不损失 X2 年化（≥36%）的前提下，通过 Dual Momentum 绝对动量过滤降低回撤（目标 -32% ~ -35%）
- Dual Momentum 仅需 close 数据，规避 high/low 数据缺失问题
- 提升 Calmar（目标 ≥1.1），Sharpe 不劣化

## 十、教训总结（供后续版本参考）

1. **因子对数据质量敏感时必须先验数据完备性：** RSRS 依赖 high/low，87.6% 数据缺失直接破坏 z-score 统计特性，触发率 36.8% 远超理论 15.87% 是数据问题的直接证据
2. **前序版本的因子结论不可无条件迁移：** X3 的"clip ±0.025 正贡献"是防守语境下的相对改善，迁移到 X4 进攻框架后变负贡献 -1.11pp
3. **触发频率是检验因子统计健康度的金标准：** 若实际触发频率远偏离理论值（如 2 倍以上），应立即怀疑数据或因子实现问题，而非调参
4. **回撤改善需配合权衡比评估：** 年化损失 / 回撤改善 > 2 即不可接受，X4 的 5.33 远超阈值
5. **数据受限时优先选择低数据依赖的替代方案：** RSRS 需 high/low，Dual Momentum 仅需 close——在数据缺失场景下应选后者
6. **N 值参数最优不能挽救方法论失败：** N=24 回撤最优 -32.09%，但年化仍 -5.79pp 低于 X2，说明数据根因不解决，调参无效
