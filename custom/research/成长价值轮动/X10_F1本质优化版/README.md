## 设计哲学

用户洞察："F1的34%→36%差距是回撤控制不足导致的，F1负责进攻，找不降收益的防回撤方法才是正确的路"

X10从F1本质研究出发，放弃X9-2的4因子复杂结构（F1+F2原版+F2''_perp+MA），回归2因子简洁设计：
1. **收益增强层**：F1 + A1斜率确认 + B2 value动量确认
2. **风险防御层**：E5持仓方向止损降仓

## 策略逻辑

### 收益增强层（F1+A1+B2）

```python
# F1信号：比价MA20方向
ratio = g_close / v_close
ratio_ma20 = ratio.rolling(20).mean()
ratio_dev = ratio / ratio_ma20 - 1
f1_signal = np.tanh(ratio_dev * 30) * 0.5
base_dir = (f1_signal > 0).map({True: 'growth', False: 'value'})

# A1: MA20斜率>0.3% + 2天确认（过滤震荡市假信号）
ma20_slope = (ratio_ma20 - ratio_ma20.shift(5)) / ratio_ma20.shift(5)
slope_ok = ma20_slope.abs() > 0.003
dir_confirmed = base_dir.where(base_dir == base_dir.shift(1), np.nan)
a1_dir = dir_confirmed.where(slope_ok, np.nan).ffill()

# B2: value方向要求价值指数20日动量>0（过滤value方向弱信号）
v_mom20 = v_close.pct_change(20)
wrong_value = (a1_dir == 'value') & (v_mom20 <= 0)
x10_dir = a1_dir.copy()
x10_dir[wrong_value] = 'growth'
```

### 风险防御层（E5/F3）

```python
# 持仓方向20日跌幅>10% → 降仓30%（趋势反转止损）
g_dd20 = g_close / g_close.shift(20) - 1
v_dd20 = v_close / v_close.shift(20) - 1
growth_stop = (x10_dir == 'growth') & (g_dd20 < -0.10)
value_stop = (x10_dir == 'value') & (v_dd20 < -0.10)
x10_wt = pd.Series(1.0, index=x10_dir.index)
x10_wt[growth_stop | value_stop] = 0.30
```

## 优化路径

```
F1纯日频 (34.47%/-45.21%) — 研究起点
    ↓
F1深度分析 — 发现3个问题：
    1. 震荡市假信号（≤7天持仓61%占比，40%胜率）
    2. value方向弱（36%胜率，-0.43%平均收益）
    3. 系统性熊市防御不足
    ↓
Round 1: A1+B2单独达标但回撤大
    - A1(斜率+2天确认) = 38.12%/-45.52%
    - B2(value动量确认) = 38.87%/-51.84%
    ↓
Round 2: D1(A1+B2组合) = 39.37%/-48.89%
    - 防御层(MA75/MA250/暴跌)都失败
    ↓
Round 3: E5(持仓止损) = 37.89%/-37.65%/Calmar1.006 ✅
    - 关键突破：基于持仓亏损的止损，而非指数位置
    ↓
Round 4: F3参数优化 = 37.26%/-34.18%/Calmar1.090 ★
    - 降仓30%比50%更好（保留更多收益）
```

## 关键洞察

1. **防御层基于指数位置会误触发**：MA75/MA250/暴跌在轮动策略中效果差
2. **基于持仓亏损的止损更有效**：只在真正趋势反转时触发
3. **F1本质是趋势跟踪**：低胜率(50.3%)+高盈亏比(2.18:1)，靠"赚大亏小"赚钱
4. **value方向是拖后腿的**：B2过滤value方向弱信号提升收益
5. **震荡市假信号是亏损源**：A1斜率确认+2天确认过滤假信号
