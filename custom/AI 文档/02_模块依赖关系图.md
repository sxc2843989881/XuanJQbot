## 三、回测调用链路

### 3.1 SMA 回测完整调用链

```
用户执行：
    python -m custom.backtests.backtest_sma_cross

调用链：
    backtest_sma_cross.py::run_backtest()
        │
        ├─→ fetch_data()                    ← baostock 获取数据
        │
        ├─→ bt.Cerebro()                    ← 创建 backtrader 引擎
        │   ├─ addstrategy(SmaCrossStrategy)  ← 来自 custom.strategies.sma_cross_strategy
        │   ├─ adddata(PandasData)
        │   ├─ addanalyzer(SharpeRatio)
        │   ├─ addanalyzer(DrawDown)
        │   └─ addanalyzer(TimeReturn)
        │
        ├─→ cerebro.run()                   ← 执行回测
        │
        └─→ 计算指标 & 打印摘要
            ├─ calc_max_drawdown()          ← 本地实现（完整周期回撤）
            ├─ annual_return
            └─ calmar = annual_return / |max_dd|
```

### 4.2 etf_rotation 子包
```
strategies/etf_rotation/
├── __init__.py            ← 包描述
├── 1_research/            ← 阶段1子目录
├── 2_backtest/            ← 阶段2子目录
```

- 策略内 4 阶段子目录：单策略专用的阶段产物（如 `etf_rotation/1_research/universe.csv`）

## 八、循环依赖检查

经分析，custom 目录内**无循环依赖**：
- `reports/` 模块自上而下单向依赖（pipeline → plotter/html_reporter → contract/metrics）
- `strategies/` 不依赖 `reports/`（解耦）
- `factors/` 不依赖任何 custom 模块（纯函数库）
- `tests/` 依赖被测模块但不被其他模块依赖

依赖方向单向清晰：**脚本层 → 策略层 → 通用模块层 → 外部依赖**。

## 九、模块耦合度评估

| 模块 | 耦合度 | 说明 |
|------|--------|------|
| `reports/contract.py` | 极低 | 仅依赖 pandas，纯数据结构定义 |
| `reports/metrics.py` | 极低 | 仅依赖 pandas/numpy，纯函数 |
| `reports/pipeline.py` | 中 | 编排多个子模块 |
| `reports/factor_plugins/` | 低 | 仅依赖 contract.base |
| `strategies/` | 低 | 自包含 |
| `factors/` | 极低 | 纯函数 |
| `backtests/` | 中 | 依赖 strategies + 外部 baostock/backtrader |

**结论**：整体耦合度良好，reports 模块通过契约+插件模式实现了与策略的完全解耦。
