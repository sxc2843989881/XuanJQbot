## 一、reports 通用报告模块（核心）⭐

### 1.1 设计哲学：契约 + 插件解耦

reports 模块的核心设计是**与策略完全解耦**：任何策略只要按数据契约（`BacktestData`）传入回测结果，即可生成统一风格的报告。

```
┌────────────────┐     ┌────────────────┐     ┌────────────────┐
│   策略层       │     │   契约层       │     │   报告层       │
│ (SMA/MACD/...) │ ──→ │ BacktestData   │ ──→ │ ReportPipeline │
│                │     │ BacktestMetrics│     │ plotter/html   │
└────────────────┘     └────────────────┘     └────────────────┘
                              ↑
                              │
                       ┌────────────────┐
                       │  FactorPlugin  │ ← 策略特定的因子子图
                       │  (可插拔插件)  │
                       └────────────────┘
```

### 1.2 数据契约：contract.py

#### BacktestData（回测数据）

@dataclass
class BacktestData:

### 1.3 指标计算：metrics.py

#### 完整周期最大回撤（用户偏好）
```python
def calc_max_drawdown(equity: pd.Series) -> float:
    """完整周期最大回撤：从上一轮历史高点到下一轮低点
    
    用户偏好：不按年切分，反映整个回测期内的最坏情况
    """
    running_max = equity.cummax()
    drawdown = (equity - running_max) / running_max
    return float(drawdown.min())
```

**关键点**：
- 使用 `cummax()` 跟踪历史高点
- 回撤 = (当前值 - 历史高点) / 历史高点
- 返回最小值（最负值），如 -0.12 表示 -12%

#### 滚动 Sharpe
```python
def calc_rolling_sharpe(returns: pd.Series, window: int = 252, ...) -> pd.Series:
    """滚动 Sharpe Ratio（默认 252 日窗口）"""
    excess = returns - rf / periods_per_year
    rolling_mean = excess.rolling(window=window, min_periods=window // 2).mean()
    rolling_std = excess.rolling(window=window, min_periods=window // 2).std()
    return (rolling_mean / rolling_std) * np.sqrt(periods_per_year)
```

    return {...}

### 1.4 因子插件机制：factor_plugins/

#### 抽象基类
```python
class FactorPlugin(ABC):
    @abstractmethod
    def plugin_name(self) -> str: ...
    
    @abstractmethod
    def prepare(self, data: BacktestData) -> BacktestData:
        """计算因子序列、信号、金叉死叉点，填充到 data"""
    
    @abstractmethod
    def factor_plot_specs(self) -> List[FactorPlotSpec]:
        """返回因子子图要绘制的因子线规格"""
    
    def signal_yticklabels(self) -> Optional[List[str]]:
        """默认 None，用通用标签"""
    
    def factor_subtitle(self) -> str:
        """默认空"""
```

#### SMA 插件实现
```python
class SmaFactorPlugin(FactorPlugin):
    def __init__(self, pfast: int = 10, pslow: int = 30):
        self.pfast = pfast
        self.pslow = pslow
    
    def prepare(self, data: BacktestData) -> BacktestData:
        # 1. 计算 SMA 快/慢线（若策略未填充）
        sma_fast = self._calc_sma(data.closes, self.pfast)
        sma_slow = self._calc_sma(data.closes, self.pslow)
        data.factors[f"SMA{self.pfast}"] = sma_fast
        data.factors[f"SMA{self.pslow}"] = sma_slow
        
        # 2. 计算信号：1=多头, -1=空头, 0=平
        signal = pd.Series(0, index=data.dates, dtype=int)
        signal[sma_fast > sma_slow] = 1
        signal[sma_fast < sma_slow] = -1
        data.signals = signal
        
        # 3. 计算金叉/死叉点（信号变化点）
        sig_diff = signal.diff().fillna(0)
        golden_cross = list(data.dates[sig_diff > 0])    # 0/-1 → +1
        death_cross = list(data.dates[sig_diff < 0])     # +1/0 → -1
        data.cross_points = (golden_cross, death_cross)
        
        return data
    
    def factor_plot_specs(self) -> List[FactorPlotSpec]:
        return [
            FactorPlotSpec(name=f"SMA{self.pfast}", label=f"SMA{self.pfast}（快线因子）",
                          color='#FF6B35', linewidth=1.3),
            FactorPlotSpec(name=f"SMA{self.pslow}", label=f"SMA{self.pslow}（慢线因子）",
                          color='#1E88E5', linewidth=1.3),
        ]
```

## 三、SMA 双均线策略实现

### 3.1 策略代码：sma_cross_strategy.py

```python
class SmaCrossStrategy(bt.Strategy):
    params = (('pfast', 10), ('pslow', 30),)
    
    def __init__(self):
        sma_fast = bt.indicators.SMA(period=self.p.pfast)
        sma_slow = bt.indicators.SMA(period=self.p.pslow)
        self.crossover = bt.indicators.CrossOver(sma_fast, sma_slow)
    
    def next(self):
        if self.position.size == 0:
            if self.crossover > 0:        # 金叉
                self.buy()
        elif self.position.size > 0:
            if self.crossover < 0:        # 死叉
                self.close()
```

**关键逻辑**：
- 仅做多（`position.size == 0` 时买入）
- 金叉买入，死叉平仓
- 满仓操作（未设仓位比例）

### 3.2 因子逻辑：factor_sma.py

```python
def calc_sma(series: pd.Series, period: int) -> pd.Series:
    """简单移动平均"""
    return series.rolling(window=period, min_periods=1).mean()

def sma_cross_signal(close: pd.Series, fast: int = 10, slow: int = 30) -> pd.Series:
    """SMA 交叉信号：1=多头, -1=空头, 0=无"""
    sma_fast = calc_sma(close, fast)
    sma_slow = calc_sma(close, slow)
    diff = sma_fast - sma_slow
    signal = pd.Series(0, index=close.index)
    signal[diff > 0] = 1
    signal[diff < 0] = -1
    return signal
```

**与策略的关系**：
- 策略用 backtrader 内置 `bt.indicators.SMA`（与因子逻辑等价）
- 因子库供非 backtrader 场景使用（如独立研究脚本）

## 八、关键设计模式总结

### 8.1 契约模式（Contract Pattern）
- `contract.py` 定义数据契约
- 策略按契约组装数据
- 报告层按契约消费数据
- **优势**：策略与报告完全解耦

### 8.2 插件模式（Plugin Pattern）
- `FactorPlugin` 抽象基类定义接口
- 具体策略实现插件（如 `SmaFactorPlugin`）
- 报告层通过插件接口调用
- **优势**：新策略只需写插件，无需改报告层

### 8.3 流水线模式（Pipeline Pattern）
- `ReportPipeline` 编排多个步骤
- 数据契约 → 指标计算 → 因子填充 → 绘图 → HTML
- **优势**：流程清晰，易于扩展

### 8.4 4 阶段生命周期模式
- 每阶段有独立目录、入口脚本、产物目录
- 阶段间通过门槛检查
- **优势**：避免跳阶段，保证研究严谨性
