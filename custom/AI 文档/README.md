## custom 目录概览

`custom/` 是 Qbot 项目中**用户私有策略与产出的根目录**，与 Qbot 开源代码完全隔离。所有新增策略、因子、回测、研究笔记均放在此目录下，**不修改** `qbot/`、`pytrader/`、`pyfunds/` 等原项目代码。

### 核心设计理念：4 阶段生命周期

策略研究必须按 4 阶段顺序推进，**不允许跳阶段**：

```
研究 (research) → 回测 (backtests) → 验证 (validation) → 实盘 (live)
   阶段 1            阶段 2             阶段 3             阶段 4
```

每阶段通过门槛才能进入下一阶段，失败回退到上一阶段。

### 顶层目录速查

| 目录 | 阶段 | 用途 |
|------|------|------|
| `research/` | 阶段 1 | 因子有效性分析、信号探索、假设检验 |
| `backtests/` | 阶段 2 | 完整策略历史回测（backtrader 引擎） |
| `validation/` | 阶段 3 | OOS、Walk-Forward、参数敏感性、稳健性检验 |
| `live/` | 阶段 4 | 模拟盘、实盘下单、监控、风控 |
| `strategies/` | 跨阶段 | 策略代码共享区（含 etf_rotation 子包） |
| `factors/` | 跨阶段 | 因子逻辑库（与策略解耦） |
| `reports/` | 通用 | 通用回测报告生成模块（与策略解耦） |
| `tests/` | 通用 | 测试脚本 |
| `notebooks/` | 通用 | 研究笔记（Jupyter） |
| `data/` | 缓存 | 本地数据缓存（默认从网络获取） |
| `output/` | 产物 | 统一输出目录（运行时生成） |
| `logs/` | 日志 | 运行日志（按类型分子目录） |
